Close a Resampling Filter Object

resamp_filter_close()

Syntax

SAL_i32 resamp_filter_close(
  SAL_resamp_filter filter_handle, /* filter handle */
  SAL_i32 flag ); /* reserved, specify 0 */

Description

The resamp_filter_close() function closes a resampling filter object.

If the filter object specified by filter_handle is currently in use by one or more active map objects, the filter object does not actually close, but is scheduled to close when the last map object that references the filter object closes.

If the filter object was created with resamp_filter_create(), the previously allocated object memory is freed after all map objects that reference the filter object are also closed.

If the filter object was created with resamp_filter_create_user_alloc(), the object memory is not freed. You are responsible for freeing the memory.

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument is detected. The function was not executed.

Create a Resampling Filter Object from a Set of Polyphase Filters

resamp_filter_create(), resamp_filter_create_user_alloc()

Syntax

SAL_i32 resamp_filter_create(
  SAL_f32 *filter_set, /* input filter set */
  SAL_i32 filter_len, /* filter length in filter_set */
  SAL_i32 num_filters, /* number of filters in _set */
  SAL_resamp_name function_name, /* resampling function name */
  SAL_i32 flag, /* reserved, specify 0 */
  SAL_resamp_filter *filter_handle, /* filter handle or NULL */
  SAL_i32 *num_bytes ); /* number of alloc. Bytes or NULL */
SAL_i32 resamp_filter_create_user_alloc(
  SAL_f32 *filter_set, /* input filter set */
  SAL_i32 filter_len, /* filter length in filter_set */
  SAL_i32 num_filters, /* number of filters in _set */
  SAL_resamp_name function_name, /* resampling function name */
  SAL_i32 flag, /* reserved, specify 0 */
  SAL_resamp_filter *filter_handle, /* filter handle, never NULL */
  void *buffer_ptr ); /* user alloc. buffer for */

Description

The resamp_filter_create() function creates a resampling filter object from a set of polyphase filters, implicitly allocating space for internal storage. You can also use this function to determine the memory requirements for explicit memory allocation.

The resamp_filter_create_user_alloc() function creates a resampling filter object from a set of polyphase filters using a user-allocated buffer for internal storage.

Parameters

filter_set

A polyphase filter set built by resamp_make_filter_set() or created manually according to the algorithm for constructing filter sets.

filter_len

Length of each interpolation filter in filter_set.

num_filters

Number of interpolation filters in filter_set.

function_name

Determines the specific resampling function that can later be invoked with this filter object. These values cannot be ORed together. Specify only one of the following values according to the resampling function you intend to call:

  SAL_RESAMP_ROCX SAL_RESAMP_MR_ROCX SAL_RESAMP_MC_ROCX
  SAL_RESAMP_COCX SAL_RESAMP_MR_COCX SAL_RESAMP_MC_COCX
  SAL_RESAMP_ZOCX SAL_RESAMP_MR_ZOCX SAL_RESAMP_MC_ZOCX
  SAL_RESAMP_RICX SAL_RESAMP_MR_RICX SAL_RESAMP_MC_RICX
  SAL_RESAMP_CICX SAL_RESAMP_MR_CICX SAL_RESAMP_MC_CICX
  SAL_RESAMP_ZICX SAL_RESAMP_MR_ZICX SAL_RESAMP_MC_ZICX

filter_handle

Filter object handle.

You can specify NULL only for the resamp_filter_create() function and only if the num_bytes value is not NULL. If the filter_handle is NULL, then the filter object is not created, but its required size is returned in num_bytes.

num_bytes

You can specify NULL only if the filter_handle value is not NULL. If non-NULL, then returns the required size of the filter object in bytes.

buffer_ptr

Must be large enough to meet the internal buffer requirements. You can determine this size in bytes by invoking resamp_filter_create() with the filter_handle parameter specified as NULL, and the num_bytes parameter specified as non-NULL.

Do not modify the content of the buffer until the filter_handle is freed by calling resamp_filter_close().

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument is detected. The function was not executed.

SAL_NOT_ENOUGH_MEMORY

Allocation failure.

Construct a Polyphase Filter Set

resamp_make_filter_set()

Syntax

SAL_i32 resamp_make_filter_set(
  SAL_f64 (*filter_func) (double), /* pointer to filter function */
  SAL_f32 *filter_set, /* output filter set */
  SAL_i32 filter_len, /* length of each filter in filter_set */
  SAL_i32 num_filters ); /* number of filters in filter_set */

Description

The resamp_make_filter_set() function constructs a polyphase filter set from a user-supplied real-valued filter function. You can use the resulting filter_set as the input to the resamp_filter_create() and resamp_filter_create_user_alloc() functions.

Filter sets are built according to the following algorithm:

fs(i,j) = filter_funci (j – L)

where filter_funci (x) = filter_func (x – i/num_filters), and:

L = floor ( (filter_len – 1) / 2)

j = 0, filter_len – 1

i = 0, num_filters – 1

Parameters

filter_func

A continuous, real-valued (typically symmetric) filter function centered about the origin and defined in a neighborhood on either side of the origin from at least -(filter_len/2 + 1) to (filter_len/2 + 1).

A careful examination of the above algorithm reveals that filter_func is effectively centered over i/num_filters when computing the ith filter in the filter set.

To maintain as much accuracy as possible during polyphase filter construction, the filter function is defined as double precision, although the former has single precision coefficients.

filter_set

Output filter set. An array with size of (num_filters x filter_len).

filter_len

Length of each interpolation filter in filter_set.

num_filters

Number of interpolation filters in filter_set.

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument is detected. The function was not executed.

Close a Reference Map Object

resamp_map_close()

Syntax

SAL_i32 resamp_map_close(
  SAL_resamp_map map_handle, /* map handle */
  SAL_i32 flag ); /* reserved, specify 0 */

Description

The resamp_map_close() function closes a resampling map object. The underlying filter object is not closed until you invoke the resamp_filter_close() function.

If the map object was created with resamp_map_create(), the memory that was previously allocated for the object is freed.

If the map object was created with resamp_map_create_user_alloc(), the object memory is not freed. You are responsible for freeing the memory.

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument is detected. The function was not executed.

Create a Resampling Map Object from a Set of Reference Map Tables

resamp_map_create(), resamp_map_create_user_alloc()

Syntax

SAL_i32 resamp_map_create(
  SAL_resamp_filter filter_handle, /* filter handle */
  SAL_f32 *map_table_set, /* input reference map set */
  SAL_i32 map_table_len, /* length of each map in map_table_set */
  SAL_i32 num_map_tables, /* no. of maps in map_table_set */
  SAL_i32 uniform_seq_len, /* length of each uniform sequence */
  SAL_i32 flag, /* boundary check flag */
  SAL_resamp_map *map_handle, /* map handle or NULL */
  SAL_i32 *num_bytes ); /* size of map object */
SAL_i32 resamp_map_create_user_alloc(
  SAL_resamp_filter filter_handle, /* filter handle */
  SAL_f32 *map_table_set, /* input reference map set */
  SAL_i32 map_table_len, /* length of each map in map_table_set */
  SAL_i32 num_map_tables, /* no. of maps in map_table_set */
  SAL_i32 uniform_seq_len, /* length of each uniform sequence */
  SAL_i32 flag, /* boundary check flag */
  SAL_resamp_map *map_handle, /* map handle */
  SAL_i32 *buffer_ptr ); /* user alloc. internal buffer */

Description

The resamp_map_create() and resamp_map_create_user_alloc() functions create a resampling map object from a set of reference map tables. The resamp_map_create() function implicitly allocates space for internal storage. The resamp_map_create_user_alloc() function uses a user-allocated buffer for internal storage.

Parameters

filter_handle

Filter handle previously returned by the resamp_filter_create() or resamp_filter_create_user_alloc() function.

map_table_set

Reference map table set. A 2-D array of reference maps of size (num_map_tables x map_table_len), where each reference map position (ref_map) is a real value, so that 0 ≤ ref_map < uniform_seq_len.

map_table_len

Length of each reference map table in map_table_set.

num_map_tables

Number of reference map tables in map_table_set.

uniform_seq_len

Length of each uniform sequence (input for OC or output for IC) indirectly addressed by the reference map values in map_table_set.

flag

Specify one of the following boundary check values:

0 (zero)

No boundary checking is performed on the reference map values in map_table_set (default). It is assumed that all reference maps are in-bounds.

SAL_RESAMP_ZERO_OUT_OF_BOUNDS

If a reference map is out-of-bounds, then a zero value is written or added to the relevant output points.

SAL_RESAMP_TAPER_OUT_OF_BOUND

If a reference map is out-of-bounds, the filter is applied as if the uniform sequence was extended infinitely in both directions with zero values. So, only the sequence in-bounds points are included in the resampling calculation.

Note 

A reference map position (ref_map) is defined as in-bound if:

floor ( (filter_len – 1) / 2) <= 

ref_map + 1/(2 * num_filters) <

uniform_seq_len – floor (filter_len / 2)

Otherwise, it is defined to be out-of-bounds.

filter_handle

Filter object handle.

num_bytes

You can specify NULL only if the map_handle value is not NULL. If non-NULL, then receives the required size of the filter object in bytes.

map_handle

Map object handle. You can specify NULL only for the resamp_map_create() function and only if the num_bytes value is not NULL. If the map_handle is NULL, then the filter object is not created, but its required size is returned in num_bytes.

buffer_ptr

Must be large enough to meet the internal buffer requirements. You can determine this size in bytes by invoking resamp_map_create() with the map_handle parameter specified as NULL.

Do not modify the content of the buffer until the map_handle is freed by calling resamp_map_close().

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument is detected. The function was not executed.

SAL_NOT_ENOUGH_MEMORY

Allocation failure.

Description

The resamp_ricx(), resamp_cicx(), resamp_zicx(), resamp_mc_ricx(), resamp_mc_cicx(), resamp_mc_zicx(), resamp_mr_ricx(), resamp_mr_cicx() and resamp_mr_zicx() functions perform single, multiple column, and multiple row input-centered (IC) 1-D resampling.

Each input point is associated with a reference map value from the specified reference map table. Each reference map value effectively selects a particular filter from the underlying polyphase filter set along with an output subsequence of the same length as the filter. The associated input point is then multiplied in turn by each filter element, and the result is added into the corresponding output point in the output subsequence.

Before calling each of these functions, you must create a map object using the resamp_map_create() or resamp_map_create_user_alloc() function, and prior to that you must create the underlying filter object by calling resamp_filter_create() or resamp_filter_create_user_alloc() with the corresponding function_name. Filter objects created for a particular function_name can only be used in the function with the matching name.

The functions sequentially access the reference map table specified by map_handle and map_table at a zero-based index map_table_start_idx. Therefore, there must be at least input_len points in the map table starting at map_table_start_idx, as indicated by the following formula:

map_table_start_idx + input_len ≤ map_table_len

The multiple column (_mc_) and the multiple row (_mr_) functions are functionally identical to their single function counterparts, except that for the same reference maps specified by map_handle, map_table, map_table_start_idx, and input_len are used repeatedly to resample the specified number of columns (_mc_) or rows (_mr_).

Parameters

map_handle

Map handle previously returned by one of the resamp_map_create(), resamp_map_create_user_alloc() functions.

map_table

Zero-based index of the reference map table to use in the resampling operation.

map_table_start_idx

Starting index into reference map table corresponding to the zero input point. Usually zero.

input

Single functions: resamp_{r|c|z}icx()

Pointer to the input sequence input_len.

Multiple column functions: resamp_mc_{r|c|z}icx()

Pointer to the input matrix of sampled columns.

Multiple row functions: resamp_mr_{r|c|z}icx()

Pointer to the input matrix of sampled rows.

output

Single functions: resamp_{r|c|z}icx()

Pointer to the output sequence that is indirectly addressed by the reference map values in map_table.

Multiple column functions: resamp_mc_{r|c|z}icx()

Pointer to the output matrix of sampled columns.

Multiple row functions: resamp_mr_{r|c|z}icx()

Pointer to the output matrix of sampled rows.

input_tcols

Total input columns (column element stride). Multiple row and multiple column functions only.

output_tcols

Total output columns (column element stride). Multiple row and multiple column functions only.

input_len

Single functions: resamp_{r|c|z}icx()

Number of points in input.

Multiple column function: resamp_mc_{r|c|z}icx()

Number of points in each input column.

Multiple row functions: resamp_mr_{r|c|z}icx()

Number of points in each input row.

num_columns

Number of input and output columns to resample. Multiple column resamp_mc_{r|c|z}icx() functions only.

num_rows

Number of input and output rows to resample. Multiple row resamp_mr_{r|c|z}icx() functions only.

flag

Specify one of the following accumulation options:

0 (zero)

Before entering the IC resampling loop, the entire output sequence is initialized to zero (default).

SAL_RESAMP_ACCUMULATE

The output sequence is not initialized. You are responsible for initializing the output sequence before calling the function.

Returns

SAL_SUCCESS

Successful execution.

SAL_INVALID_ARGUMENT

An invalid argument was detected. The function was not executed.

Description

The resamp_rocx(), resamp_cocx(), resamp_zocx(), resamp_mc_rocx(), resamp_mc_cocx(), resamp_mc_zocx(), resamp_mr_rocx(), resamp_mr_cocx(), and resamp_mr_zocx() functions perform single, multiple column, and multiple row output-centered (OC) 1-D resampling.

Each output point is a associated with a reference map value from a specified reference map table. Each reference map value effectively selects a particular filter from the underlying polyphase filter set along with an input subsequence of the same length as the filter. The output value is then calculated as the dot-product between the filter and the input subsequence and is written to or added into the associated output point, depending on the value of the accumulation flag.

Before calling each of these functions, you must create a map object using the resamp_map_create() or resamp_map_create_user_alloc() function, and prior to that you must create the underlying filter object by calling resamp_filter_create() or resamp_filter_create_user_alloc() with the corresponding function_name. Filter objects created for a particular function_name can be used only in the function with the matching name.

The functions sequentially access the reference map table specified by map_handle and map_table at a zero-based index map_table_start_idx. Therefore, there must be at least output_len points in the map table starting at map_table_start_idx, as indicated by the following formula:

map_table_start_idx + output_len ≤ map_table_len

The multiple column (_mc_) and the multiple row (_mr_) functions are functionally identical to their single function counterparts, except that the same reference maps specified by map_handle, map_table, map_table_start_idx, and output_len are used repeatedly to resample the specified number of columns (_mc_) or rows (_mr_).
