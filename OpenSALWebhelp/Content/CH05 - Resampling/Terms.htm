Resampling Terms and Definitions

To understand SAL resampling functions, you must be familiar with:

  • Output-centered resampling algorithms
  • Input-centered resampling algorithms
  • Uniform sequences
  • Reference sequences
  • Reference points
  • Reference maps
  • Reference map tables
  • Polyphase filter sets

Input-Centered and Output-Centered Resampling 

The output-centered (OC) resampling routines handle uniformly sampled input sequences. Input-centered (IC) routines handle uniformly sampled output sequences.

When both sequences are uniformly sampled, use OC or IC. However, depending on the processor architecture, OC implementations are more efficient.

If neither the input, nor the output sequence is uniformly sampled:

  1. Use an IC routine to form a uniformly sampled intermediate sequence.
  2. Use the intermediate sequence as an input to an OC routine to produce the required non-uniformly sampled output sequence.

Output-Centered Resampling Algorithms

An OC resampling algorithm computes an output sequence from a uniformly-spaced input sequence. This is done by:

  1. Centering a continuous, real-valued (typically symmetric) filter function over (or near) each output point
  2. Sampling the filter so that the resulting discrete filter coefficients line up exactly with some fixed number (m) of input points on either side of the output (reference) point

This discrete filter is then applied to the surrounding input points in a dot-product calculation, the result of which is written or added to the designated output point. This process is repeated for each point in the output sequence.

Input-Centered Resampling Algorithms

An IC resampling algorithm computes a uniformly-spaced output sequence from an input sequence. This is done by:

  1. Centering a continuous, real-valued (typically symmetric) filter function over (or near) each input point
  2. Sampling the filter so that the resulting discrete filter coefficients line up exactly with some fixed number (m) of output points on either side of the input (reference) point

This discrete filter is then applied to the surrounding output points by multiplying each of the m filter coefficients by the designated input point and adding each of the results to the m corresponding output points, thereby updating the m output points (one per filter coefficient).

Uniform Sequences

The uniform sequence is the uniformly-spaced sequence (sampled at a uniform spacing t = 1), over which the resampling filter is applied.

  • For OC routines, this is the input sequence.
  • For IC routines, this is the output sequence.

It is assumed that t is equal to 1 for all the routines in this specification, so that the entire sampling space ranges from 0 to N–1 units, where N is the number of points in the uniform sequence.

Reference Sequences

The reference sequence is the sequence that is being accessed (walked) sequentially during a particular resampling routine.

  • For OC routines, this is the output sequence.
  • For IC routines, this is the input sequence.

The reference sequence does not have to be uniformly-spaced.

Reference Points

The reference point is the current point in the reference sequence that is being computed (OC) or operated on (IC) routines.

Reference Maps

A reference map is the non-integral position of a given reference point within the uniform sequence.

  • For OC routines, each reference map is the position of a unique output point within the uniformly-spaced input sequence.
  • For IC routines, each reference map is the position of a unique input point within the uniformly-spaced output sequence.

The integer part of a reference map identifies the contiguous subsequence within the uniform sequence over which to apply the interpolation filter. The fractional part of a reference map determines the phase of the discretely sampled filter so that the filter coefficients line up with their corresponding points in the uniform sequence.

Reference Map Tables

A reference map table is a table of reference maps where each table entry corresponds to a unique reference point within the reference sequence. During a resampling routine, a reference map table is sequentially accessed to acquire the position of each reference point within the uniform sequence.

  • For the OC algorithm, the ith table entry corresponds to the ith output point.
  • For the IC algorithm, the ith table entry corresponds to the ith input point.

A reference map is considered out-of-bounds if the application of a complete m-point filter will result in accesses beyond either end of the uniform sequence (the input sequence for the OC algorithm or the output sequence for the IC algorithm). By default, SAL does not check for out-of-bounds reference maps. SAL supports two boundary checking options for handling out-of-bounds reference maps:

  • The Zero option applies a filter of all zeros.
  • For the OC routines, this option causes a zero to be written (or added) to the current reference point.
  • For the IC routines, this option leaves the partially computed output sequence unchanged.
  • The Taper option allows for partial application of the filter based on the number of valid points in the uniform sequence that actually fall under the filter. Filter points that correspond to invalid (beyond either end) input (OC) or output (IC) points are not included in the calculation. If all points are invalid, the behavior is the same as if the Zero option had been specified.

Polyphase Filter Sets

For SAL resampling functionality, a polyphase filter set is defined as a collection of n discrete filters of equal length m, where each filter is sampled from the same continuous real-valued filter function at the uniform time interval t = 1.

Each of the m filters represents a different discrete phase of the filter function that corresponds to the centers of the n unique phase intervals of width t/n = 1/n. Therefore, adjacent filters within the set are phase-shifted by exactly 1/n from one another.

During a resampling operation, a particular discrete filter is selected from the set based on the phase interval that the current reference point falls within (determined by its associated reference map).

The center of the continuous filter function ƒ(x) is defined as x = 0. The center of each discrete filter in the set should correspond to the center of its phase interval, and the discrete filter coefficients should line up with the m surrounding points in the uniform sequenceover which the filter is applied. This ensures that the selected filter is within 1/(2n) units from the reference point. Therefore, by increasing the number of filters in the set, this error can be made arbitrarily small (limited by the rounding error of the computer).
