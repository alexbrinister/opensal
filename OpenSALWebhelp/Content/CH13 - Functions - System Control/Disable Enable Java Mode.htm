Disable and Enable Java Mode on AltiVec Processors

sal_java_mode_disable(), sal_java_mode_enable()

Syntax

void sal_java_mode_disable ( void );

void sal_java_mode_enable ( void );

Description

If you work on non-MCexec platforms, invoke sal_java_mode_disable() once in an application before starting SAL processing to place the processor in non-Java® mode. Do not toggle between Java and non-Java modes.

Note 
The sal_java_mode_disable() and sal_java_mode_enable() functions are not required in applications that run on MCexec. The G4 and G4A2 are always in non-Java mode on the MCexec platform.

In the non-Java/non-IEEE/non-C9X mode, gradual underflow is not performed. Instead, any instruction that would have produced a denormalized result in Java mode, substitutes a correctly signed zero (+/– 0.0) as the final result. Also, denormalized input operands are flushed to the correctly signed zero before being used by the instruction.

The sal_java_mode_disable() function sets bit 15 in VSCR to 1 (VSCR[NJ]=1).

The sal_java_mode_enable() function clears bit 15 in VSCR to 0 (VSCR[NJ]=0).

No action is performed if:

  • sal_java_mode_disable() is called when the processor is already in non-Java mode
  • sal_java_mode_enable() is called when the processor is already in Java mode
