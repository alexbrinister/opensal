Fill Cache Buffer With Zeros

cache_zero_1d(), cache_zero_2d()

Syntax

void cache_zero_1d(
  char *pData, /* pointer to the data (vector) */
  SAL_i32 nbytes, /* number of bytes to zero */
  SAL_i32 channel, /* reserved, specify 0 */
  SAL_i32 flag ); /* reserved, specify 0 */
void cache_zero_2d(
  char *pData, /* pointer to the data (matrix) */
  SAL_i32 tcols, /* column stride (in bytes) */
  SAL_i32 nr, /* number of rows to zero */
  SAL_i32 nc, /* row length (in bytes) */
  SAL_i32 channel, /* reserved, specify 0 */
  SAL_i32 flag ); /* reserved, specify 0 */

Description

The cache_zero_1d() and cache_zero_2d() functions help to avoid the SMQ stall and improve performance by zeroing a vector or a matrix in the L1 or L2 data cache.

Note 

Use these functions with caution; they replace the contents of the buffer with zeros. Do not use these functions to load an output buffer that is also used as an input buffer into the L1 cache. Instead, use cache_load_1d() and cache_load_2d().

Use SAL cache management functions only with PowerPC 7445/7447 or with non-SAL-based data processing. If you use other than PPC 7445/7447 hardware or SAL functions for data processing, these functions can damage the performance of your application.
