Migrating from Single-Core to Multi-Core

To change an application with SAL calls to use multi-core enabled SAL, you must:

  1. Add the sal_create(), sal_init(), and sal_destroy() functions to your application.

    If you want to go back to using single-core SAL, you do not have to remove these functions from your code. These functions enable the multi-core versions of the SAL functions only when your application defines SAL_ENABLE_MULTICORE.

  2. Compile your application with SAL_ENABLE_MULTICORE defined in the appropriate places; for example, in your makefile or in an application source or header file, but before including the sal.h header file.
  3. In addition to linking with the SAL or the CSAL libraries, link your application with the MCSAL performance library or development library.
