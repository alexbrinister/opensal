Multi-Core SAL Macros

Because single-core and multi-core versions of SAL functions use the same names, you do not have to change any of your legacy code. Instead, you can use macros at compile time to specify whether to call single-core or multi-core versions of each function.

SAL_ENABLE_MULTICORE

At compile time, your application can define the SAL_ENABLE_MULTICORE macro to use the multi-core versions of the SAL functions. If you do not define the SAL_ENABLE_MULTICORE macro, your application calls the single-core version of the SAL functions.

SAL_SINGLE

If your application defines the SAL_ENABLE_MULTICORE macro, but you want to use the single-core version of a particular SAL function invocation, you can qualify the SAL function name with the SAL_SINGLE macro. For example:

SAL_SINGLE(vaddx)( A, 1, B, 1, C, 1, n_elems, SAL_NNN);
