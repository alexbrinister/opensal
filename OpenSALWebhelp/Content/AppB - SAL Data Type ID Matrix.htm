SAL Data Type ID Matrix

As of SAL version 10.0, Mercury has redefined how data types are identified. Table lists the new type IDs and the old type IDs.

Table 12-2: New SAL Type IDs versus Old Type IDs

New Type ID Old Type ID
SAL_char char
SAL_i8 signed char
SAL_i16 short
SAL_i32 int, long
SAL_i64 long long
SAL_ui8 unsigned char
SAL_ui16 unsigned short
SAL_ui32 unsigned int, unsigned long
SAL_ui64 unsigned long long
SAL_f32 float
SAL_f64 double
SAL_cf32 COMPLEX, COMPLEX_F32
SAL_cf64 DOUBLE_COMPLEX, COMPLEX_F64
SAL_zf32 COMPLEX_SPLIT, COMPLEX_SPLIT_F32
SAL_zf64 DOUBLE_COMPLEX_SPLIT, COMPLEX_SPLIT_F64
SAL_ci8 COMPLEX_S8
SAL_ci16 COMPLEX_S16
SAL_ci32 COMPLEX_S32
SAL_cui8 COMPLEX_U8
SAL_cui16 COMPLEX_U16
SAL_cui32 COMPLEX_U32
SAL_zi8 COMPLEX_SPLIT_S8
SAL_zi16 COMPLEX_SPLIT_S16
SAL_zi32 COMPLEX_SPLIT_S32
SAL_zui8 COMPLEX_SPLIT_U8
SAL_zui16 COMPLEX_SPLIT_U16
SAL_zui32 COMPLEX_SPLIT_U32

You can continue to use the old type IDs in your application code, and you can mix type IDs without penalty. However, going forward, Mercury recommends that you to adopt the new type IDs.
