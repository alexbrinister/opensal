In addition, SAL defines four additional data types for complex numbers:

SAL_cf32 Single-precision interleaved complex
SAL_zf32 Single-precision split complex
SAL_cf64 Double-precision interleaved complex
SAL_zf64 Double-precision split complex

Interleaved Complex

Interleaved complex numbers are organized so that the real and imaginary parts are contiguous. For example, the following four complex numbers, represented in (real, imaginary) notation:

(0,1),(2,3),(4,5),(6,7)

Are stored contiguously in memory, similar to:

0 1 2 3 4 5 6 7

A vector of interleaved complex-typed elements can be initialized with the above data, as in:

SAL_cf32 array[4];

for (i=0; i<4; i++) {

array[i].real = i*2;

array[i].imag = (i*2) + 1;

}

Subsequently, a SAL function can be called, as in:

extern void sal_func ( SAL_cf32 *ptr, SAL_i32 stride, SAL_i32 n, SAL_i32 x );

sal_func ( array, 2, 4, SAL_N );

Note 
For interleaved complex-typed elements, the unit stride is 2.

Split Complex

The split complex type is represented by an ordered pair of pointers: the first pointing to an array containing the real parts, and the second pointing to a separate array that contains the imaginary parts. The pairing of real and imaginary parts (those at the same index into their respective arrays) conceptually constitutes the individual split complex-typed elements.

For example, the following four complex numbers represented in (real, imaginary) notation:

(0,1),(2,3),(4,5),(6,7)

Are stored in two memory blocks, each contiguous within itself, as in:

0 2 4 6 ... 1 3 5 7

A vector of split complex-typed elements can be initialized with the above data, as in:

SAL_f32 real_array[4], imag_array[4];

SAL_zf32 array;

array.realp = real_array;

array.imagp = imag_array;

for (i=0; i<4; i++) {

array.realp[i] = i*2;

array.imagp[i] = (i*2) + 1;

}

Subsequently, a SAL function can be called, as in:

extern void sal_func ( SAL_zf32 *ptr, SAL_i32 stride, SAL_i32 n, SAL_i32 x );

sal_func ( &array, 1, 4, SAL_N );

Note 

Unit element stride for split complex-typed elements is 1. The stride applies to the real and imaginary part arrays.
