Vector Real to Short Convert

vfixx(), vfixrx(), vfixux()

Synopsis

void vfixx(
  SAL_f32 *A, /* input vector */
  SAL_i32 i, /* address stride for A */
  SAL_i16 *C, /* 4-byte-aligned output vector */
  SAL_i32 k, /* address stride for C */
  SAL_i32 n, /* real input count, must be even */
  SAL_i32 flag ); /* ESAL flag */
void vfixrx(
  SAL_f32 *A, /* input vector */
  SAL_i32 i, /* address stride for A */
  SAL_i16 *C, /* 4-byte-aligned output vector */
  SAL_i32 k, /* address stride for C */
  SAL_i32 n, /* real input count, must be even */
  SAL_i32 flag ); /* ESAL flag */
void vfixux(
  SAL_f32 *A, /* input vector */
  SAL_i32 i, /* address stride for A */
  SAL_ui16 *C, /* 4-byte-aligned output vector */
  SAL_i32 k, /* address stride for C */
  SAL_i32 n, /* input count, must be even */
  SAL_i32 flag ); /* ESAL flag */

Description

The vfixx(), vfixrx(), and vfixux() functions convert real values from vector A to 16-bit signed and unsigned integers.

 

The vfixx(), vfixrx(), and vfixux() functions are not recommended for new application development.

Use the newer vconvert_f32_s16x() function for vector conversion with unit strides.

Use the newer vconvert_f32_u16x() function with scale=NULL, bias=NULL, and rfflag=SAL_ROUND_ZERO.

The vfixx() and vfixux() functions ignore fractional parts of vector A and rounds toward zero. The vfixrx() function rounds results according to the current processor rounding mode.

These functions operate on two real inputs at a time, converting both to integers, and packing the integers into a 32-bit location. The functions convert input values ranging from –32,768 to 32,767, inclusive. Results for input values outside of this range are undefined.
