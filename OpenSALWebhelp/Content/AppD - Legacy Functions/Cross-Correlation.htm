Cross-Correlation

ccorx(), ccorfx(), ccortx()

Synopsis

#include <sal.h>
void ccorx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *b, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a and b */
  SAL_i32 flag ); /* ESAL flag */
    /* n ≥ m */
    /* Requires FFT weights array */
void ccorfx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *b, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a and b */
  SAL_i32 flag ); /* ESAL flag */
    /* n ≥ m */
    /* Requires FFT weights array */
void ccortx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *b, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a */
  SAL_i32 flag ); /* ESAL flag */

Description

The ccorx(), ccorfx(), and ccortx() functions perform linear cross-correlation of vectors a and b.

The general call for cross-correlation, ccorx(), uses frequency-domain or time-domain correlation. The ccorx() function evaluates m and n to select the domain that yields the best execution time.

The ccorfx() function explicitly invokes frequency-domain correlation. Frequency-domain correlation does not preserve vectors a and b. As the formula implies and as required for frequency-domain processing, the length of input vectors a and b must be greater than or equal to r, where r satisfies the following requirements:

r is a power of 2

r ≥ m + n - 1

r ≥ 16

Because frequency-domain correlation does not preserve the input vectors, calling functions must assume that vectors a and b were modified by ccorx().

The ccortx() function explicitly invokes time-domain correlation and preserves vectors a and b. When m and n are small, ccortx() is faster than ccorfx().
