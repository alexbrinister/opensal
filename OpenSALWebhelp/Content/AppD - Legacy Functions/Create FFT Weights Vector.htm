Create Fourier Transform Weights Vector

fftwtsx(), fftwtsdx()

Synopsis

#include <sal.h>
void fftwtsx(
  SAL_cf32 *c, /* output vector */
  SAL_i32 m, /* divisions of the complex unit circle */
  SAL_i32 n, /* complex output count */
  SAL_i32 flag ); /* ESAL flag */
  /* m is a power of 2 */
  /* c must be 8-byte-aligned */
void fftwtsdx(
  SAL_cf64 *c, /* output vector */
  SAL_i32 m, /* divisions of the complex unit circle */
  SAL_i32 n, /* double complex output count */
  SAL_i32 flag ); /* ESAL flag */
  /* m is a power of 2 */

Description

The fftwtsx() and fftwtsdx() functions build a table of complex exponentials, or weights array, for the Fourier transform and windowing functions that require a weights array.

Note

Note 
The fftwtsx() and fftwtsdx() functions are not recommended for new application development. Use the newer fft_setup[d]() and fft_free[d]() functions.

Before calling the fftwtsx() or fftwtsdx() function, the program must call fftwtsx() and fftwtsdx() at least once. Only one weights array can be active at a time. To supersede an existing weights array, call fftwtsx() or fftwtsdx() again.

A single call to fftwtsx() or fftwtsdx() can serve a series of Fourier transform or windowing functions if m and n are large enough for each function called and the weights array is preserved. Functions that use the weights array automatically adjust strides through the array when the table has greater resolution, or a larger m, than required.

Count m represents the number of divisions of the complex unit circle and specifies the largest number of elements that can be processed by a subsequent frequency-domain function. Count m must be a power of 2 and must equal or exceed the number of elements that the functions process using the weights array.

Count n specifies the actual number of weights stored in array c. In general, m should equal n, but because some frequency-domain functions require fewer weights than the number of elements processed, n can sometimes be smaller than m.

When allocating memory for building a weights array, keep in mind that fftwtsx(1) and fftwtsdx() generate complex numbers, so allocate two sizeof(float) words per element.
