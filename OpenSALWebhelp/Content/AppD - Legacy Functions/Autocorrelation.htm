Autocorrelation

acorx(), acorfx(), acortx()

Synopsis

#include <sal.h>
void acorx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a */
  SAL_i32 flag ); /* ESAL flag */
    /* n x m */
    /* Requires FFT weights array */
void acorfx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a */
  SAL_i32 flag ); /* ESAL flag */
    /* n x m */
    /* Requires FFT weights array */
void acortx(
  SAL_f32 *a, /* input vector */
  SAL_f32 *c, /* output vector */
  SAL_i32 m, /* output count (lags) */
  SAL_i32 n, /* input count from a */
  SAL_i32 flag ); /* ESAL flag */

Description

The acorx(), acorfx(), and acortx() functions perform linear autocorrelation of vector a.

The general call for autocorrelation, acorx(), invokes time- or frequency-domain correlation.The acorx() function evaluates m and n to choose the domain that yields the best execution time. In general, the time-domain produces better execution times when m and n are small.

The acorfx() function explicitly invokes frequency-domain correlation. Frequency-domain correlation does not preserve vector a. When frequency-domain processing is predetermined to be the best method, acorfx() performs better than acorx(). As the formula implies and as required for frequency-domain processing, the length of input vector a must be greater than or equal to r, where r satisfies all of these requirements:

r is a power of 2

r ≥ m + n – 1

r ≥ 16

Because frequency-domain correlation does not preserve the input vector, calling functions must assume that vector a was modified by acorx().

The acortx() function explicitly invokes time-domain correlation and preserves vector a. When m and n are small, acortx() is faster than acorfx().
