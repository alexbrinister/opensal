Complete Setup with User-Defined Memory Allocation

fft_setup_user_alloc()

Complete Setup with User-Defined Memory Allocation

Syntax

void fft_setup_user_alloc(
  SAL_ui32 log2n, /* base 2 exponent of element count */
  SAL_i32 flag, /* options */
  FFT_setup *setup, /* specifications (returned) */
  void *buffer ); /* buffer pointer, owned by FFT structure */

Description

The fft_setup_user_alloc() function completes setup for FFTs after you have allocated memory. Use this function if you do not want fft_setup() to allocate memory. Use fft_setup_user_alloc() in conjunction with fft_setup(), but with the setup parameter in those functions set to NULL.

After processing, free the allocated memory using the fft_free() function.

Parameters

log2n

A base-2 exponent that represents the number of divisions of the complex unit circle and specifies the largest power of two that can be processed by a subsequent frequency-domain function. It must have the same value that is passed as log2n to fft_setup(). It must equal or exceed the largest power of 2 that any subsequent function processes using the weights array.

flag

Specifies options. Must have the same value that is passed as flag to fft_setup(). Any combination of the following radix options can be ORed together:

0

Radix 2 weights array.

FFT_RADIX3

Radix 3 weights array. The array can also be used by radix 2 functions.

FFT_RADIX5

Radix 5 weights array. The array can also be used by radix 2 functions.

FFT_REAL_ONLY

Use for optimal performance in one-dimensional, real-valued FFTs. You cannot use the resulting FFT setup in any other FFTs.

If both FFT_RADIX3 and FFT_RADIX5 are specified, the array can be used by radix 2, radix 3, and radix 5 functions.

setup

Points to a caller-allocated structure which defines the characteristics of the weights array to any functions that subsequently use the array. The structure is filled in by the function and subsequently passed as an argument to FFT and windowing functions.

buffer

A pointer to a buffer at least as large as the value returned in the nbytes parameter in a previous call to fft_setup().
