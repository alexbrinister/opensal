Out-of-place Real Fourier Transform, Interleaved Complex Format

fft_ropx(), fft_ropdx(), fft_roptx(), fft_roptdx()

Syntax

void fft_ropx(
  FFT_setup *setup, /* weights array */
  SAL_f32 *A, /* input vector (destroyed) */
  SAL_i32 i, /* element stride for A */
  SAL_f32 *C, /* output vector */
  SAL_i32 k, /* element stride for C */
  SAL_ui32 log2N, /* base 2 exponent of element count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft_ropdx(
  FFT_setupd *setup, /* weights array */
  SAL_f64 *A, /* input vector (destroyed) */
  SAL_i32 i, /* element stride for A */
  SAL_f64 *C, /* output vector */
  SAL_i32 k, /* element stride for C */
  SAL_ui32 log2N, /* base 2 exponent of element count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft_roptx(
  FFT_setup *setup, /* weights array */
  SAL_f32 *A, /* input vector (preserved) */
  SAL_i32 i, /* element stride for A */
  SAL_f32 *C, /* output vector */
  SAL_i32 k, /* element stride for C */
  SAL_f32 *T, /* temporary vector */
  SAL_ui32 log2N, /* base 2 exponent of element count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft_roptdx(
  FFT_setupd *setup, /* weights array */
  SAL_f64 *A, /* input vector (preserved) */
  SAL_i32 i, /* element stride for A */
  SAL_f64 *C, /* output vector */
  SAL_i32 k, /* element stride for C */
  SAL_f64 *T, /* temporary vector */
  SAL_ui32 log2N, /* base 2 exponent of element count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */

Description

The fft_ropx(), fft_ropdx(), fft_roptx(), and fft_roptdx() functions compute an out-of-place real discrete Fourier transform, from:

  • The time domain to the frequency domain (forward)

    - or - 

  • The frequency domain to the time domain (inverse)

Forward transforms read real input and write packed complex output. Inverse transforms read packed complex input and write real output. As a result of packing the frequency-domain data, time-domain data and its equivalent frequency-domain data have the same storage requirements. The fft_ropx() and fft_ropdx() functions destroy the input vector, while fft_roptx() and fft_roptdx() preserve the input vector.

Parameters

setup

Points to a structure initialized by a prior call to an FFT weights array function, fft1d_setup(). The value supplied as parameter N of the setup function must equal or exceed the value supplied as parameter N of the transform function.

i

Specifies an address stride through input vector a.

k

Specifies an address stride through output vector C. Specifying 1 process every element, specifying 2 processes every other element, and so on.

T

For fft_roptx() and fft_roptdx(), this is a temporary vector used for storing interim results. Vector T must be large enough to hold the element count specified by parameter N.

log2N

The base 2 exponent of the number of elements to process. For example, to process 1024 elements, specify 10 for parameter log2N.

f

A forward/inverse directional flag that must specify one of the following:

FFT_FORWARD

Forward transform (real to packed complex).

FFT_INVERSE

Inverse transform (packed complex to real).

Results are undefined for other values of f.
