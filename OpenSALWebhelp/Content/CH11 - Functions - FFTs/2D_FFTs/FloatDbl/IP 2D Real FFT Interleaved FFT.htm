In-Place Two-Dimensional Real Fourier Transform, Interleaved Complex Format

fft2d_ripx(), fft2d_ripdx(), fft2d_riptx(), fft2d_riptdx()

Syntax

void fft2d_ripx(
  FFT_setup *setup, /* weights array */
  SAL_f32 *C, /* input/output matrix */
  SAL_i32 k, /* row stride for C */
  SAL_i32 l, /* column stride for C */
  SAL_ui32 N, /* base 2 exponent of column count */
  SAL_ui32 M, /* base 2 exponent of row count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft2d_ripdx(
  FFT_setupd *setup, /* weights array */
  SAL_f64 *C, /* input/output matrix */
  SAL_i32 k, /* row stride for C */
  SAL_i32 l, /* column stride for C */
  SAL_ui32 N, /* base 2 exponent of column count */
  SAL_ui32 M, /* base 2 exponent of row count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft2d_riptx(
  FFT_setup *setup, /* weights array */
  SAL_f32 *C, /* input/output matrix */
  SAL_i32 k, /* row stride for C */
  SAL_i32 l, /* column stride for C */
  SAL_f32 *D, /* temporary matrix */
  SAL_ui32 N, /* base 2 exponent of column count */
  SAL_ui32 M, /* base 2 exponent of row count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */
void fft2d_riptdx(
  FFT_setupd *setup, /* weights array */
  SAL_f64 *C, /* input/output matrix */
  SAL_i32 k, /* row stride for C */
  SAL_i32 l, /* column stride for C */
  SAL_f64 *D, /* temporary matrix */
  SAL_ui32 N, /* base 2 exponent of column count */
  SAL_ui32 M, /* base 2 exponent of row count */
  SAL_i32 f, /* directional flag */
  SAL_i32 eflag ); /* ESAL flag */

Description

The fft2d_ripx(), fft2d_ripdx(), fft2d_riptx(), and fft2d_riptdx() functions compute an in-place real discrete Fourier transform, from the spatial domain to the frequency domain (forward) or from the frequency domain to the spatial domain (inverse).

Forward transforms read real input and write packed complex output. Inverse transforms read packed complex input and write real output. As a result of packing the frequency-domain data, spatial-domain data and its equivalent frequency-domain data have the same storage requirements.

Parameters

setup

Points to a structure initialized by a prior call to FFT weights array function, fft2d_setup() . The value supplied as parameter N of the setup function must equal or exceed the value supplied as parameter N or M, whichever is larger, of the transform function.

k

Specifies a stride across each row of matrix C. Specify 1 to process every element across each row; specify 2 to process every other element across each row, and so on.

l

Specifies a column stride for matrix C and, unless matrix C is a submatrix, should be allowed to default. Parameter l can be defaulted by specifying zero.

The default stride equals the row stride, parameter k, multiplied by the column count, parameter N. So, if k is 1 and l is zero, every element of matrix C is processed. If k is 2 and l is zero, every other element of each row is processed.

If not zero, parameter l represents the distance between each row of matrix C. For example, if parameter l is 1024, element 1024 of matrix C equates to element (1,0), element 2048 equates to element (2,0), and so on.

D

For fft2d_riptx() and fft2d_riptdx(), parameter D is a temporary matrix for storing interim results. Matrix D must be large enough to hold the element count as specified by parameters N and M.

N and M

Parameter N is the base 2 exponent of the number of columns to process for each row. Parameter M is the base 2 exponent of the number of rows to process. For example, to process 64 rows of 128 columns, specify 7 for parameter N and 6 for parameter M.

f

A forward/inverse directional flag that must specify one of the following:

FFT_FORWARD

Forward transform (real to packed complex).

FFT_INVERSE

Inverse transform (packed complex to real).

Results are undefined for other values of f.

For best performance, specify row strides of 1 and column strides of 0.
