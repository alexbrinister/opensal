Get Number of Bytes for Temp FFT Buffer

fft_setup_get_temp_n_bytes(), fft_setupd_get_temp_n_bytes()

Syntax

int fft_setup_get_temp_n_bytes(

FFT_setup *setup,

int *p_n_bytes,

const int flag

);

int fft_setupd_get_temp_n_bytes (

FFT_setupd *setupd,

int *p_n_bytes,

const int flag

);

Description

The fft_setup_get_temp_n_bytes() and fft_setupd_get_temp_n_bytes() functions get the number of bytes required by an FFT temporary buffer. For an FFT function that requires a temporary buffer, you can use this function to determine how large a buffer to allocate. The number of bytes returned can include additional memory to meet alignment requirements. If SAL_ENABLE_MULTICORE is defined, the size returned will be sufficient for a temporary buffer for each MCSAL thread.

Parameters

h_sal input

The created FFT_setup handle.

p_n_bytes output

The number of bytes required by the FFT temporary buffer function.

flag input

Reserved.

Return Codes

SAL_SUCCESS

Successful operation. If the operation is not successful, an error code is returned.
