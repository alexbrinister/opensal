Create Complex Exponentials for One-Dimensional Fourier Transforms

fft1d_setup(), fft1d_setupd()

Syntax

SAL_i32 fft1d_setup (
  const SAL_fft1d_func fft_type,
  const SAL_i32 log2n,
  const SAL_i32 flag,
  FFT_setup *p_setup );
SAL_i32 fft1d_setupd (
  const SAL_fft1d_funcd fft_type,
  SAL_i32 log2n,
  SAL_i32 flag,
  FFT_setupd *p_setup );

Description

The fft1d_setup() and fft1d_setupd() functions acquire and configure the resources required to perform one-dimensional FFT calculations, including tables of complex exponentials (weights arrays) for use by a specific FFT routine.

The fft1d_setup() and fft1d_setupd() routines use an enumerated type argument, whose enumerators represent the individual inner loop routines. This argument refines and optimally configures the resources. Therefore, memory is not allotted to weights arrays, index arrays, and temporary buffers that might never be used.

log2n, log2nr, log2nc

The base-2 exponent of the FFT size. The setup function uses this argument to allocate storage for weights and index arrays, and for any temporary buffers required by the implementation. It also populates these arrays with weights and index values required by the specified FFT routine.

The log2n value specified to the setup routine must also be specified when invoking the actual FFT routine.

p_setup output

A setup handle that points to an opaque structure that is allocated and populated by the setup routine and contains (among other things) pointers to weights and index arrays.

The underlying structure is completely filled in by the function, and should be, subsequently, passed as an argument to the same FFT routine specified in the fft_type parameter.

flag input

Reserved; specify SAL_RESERVED_FLAG.

SAL_INVALID_ARGUMENT

An invalid argument was passed to the function.

SAL_NOT_ENOUGH_MEMORY

Memory was insufficient for the function to perform all of its operations.

Function execution failed due to an unknown/uncategorized error.
