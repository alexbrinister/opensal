Initialize an FFT Setup Handle

fft_setup_init(), fft_setupd_init()

Syntax

int fft_setup_init (

FFT_setup *p_setup,

const int flag

);

 

int fft_setupd_init (

FFT_setupd *p_setup,

const int flag

);

Description

The fft_setup_init()and fft_setupd_init() functions initialize a previously created, but not initialized, FFT setup handle. These functions initialize the FFT setup handle to perform FFT calculations based on the FFT type and size that were specified when the FFT setup handle was created. This process includes initializing tables of complex exponentials (weights arrays) for use by the specific FFT routine.

If the application has not provided a buffer for setup, using fft_setup[d]_set_buffer() and then fft_setup[d]_init() allocates the memory necessary to complete the setup process before initialization.

Dividing the setup and initialization processes allows you to invoke optional functions to control the FFT setup process. For example:

  • To determine the number of bytes required by FFT setup for tables, your application can use fft_setup[d]_get_n_bytes() to query the FFT setup handle returned from fft1d_setup[d]_create or fft2d_setup[d]_create.
  • To provide a buffer to the FFT setup handle for this purpose, your application can use fft_setup[d]_set_buffer().

Parameters

p_setup modified

The previously created FFT setup handle.

flag input

Reserved. Instead, pass SAL_RESERVED_FLAG.

Return Codes

SAL_SUCCESS

The function executed without errors.

SAL_INVALID_FLAG

An invalid flag was passed to the function.

SAL_INVALID_HANDLE

An invalid or null handle pointer was passed to the function.

SAL_INVALID_ARGUMENT

An invalid argument was passed to the function.

SAL_NOT_ENOUGH_MEMORY

Memory was insufficient for the function to perform all of its operations.

SAL_FAILURE

Function execution failed due to an unknown/uncategorized error.
