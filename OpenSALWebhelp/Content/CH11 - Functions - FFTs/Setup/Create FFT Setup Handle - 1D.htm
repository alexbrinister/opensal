Create a 1-D Un-initialized FFT Setup Handle

fft1d_setup_create(), fft1d_setupd_create()

Syntax

int fft1d_setup_create (

const SAL_fft1d_func fft_type,

const int log2n,

const int flag,

FFT_setup *p_setup

);

 

int fft1d_setupd_create (

const SAL_fft1d_funcd fft_type,

const int log2n,

const int flag,

FFT_setupd *p_setup

);

Description

The fft2d_setup_create() and fft2d_setupd_create() functions create un-initialized FFT setup handles. These functions take the same arguments as the fft1d_setup() and fft1d_setupd() functions. However, these functions defer creating the tables of complex exponentials (weights arrays) for use by a specific FFT routine until the FFT setup handle is initialized by fft_setup_init() or fft_setupd_init().

This postponement allows you to invoke functions between creation and initialization to control the FFT setup process. For example:

  • To determine the number of bytes that FFT setup requires for tables, your application can use an fft_setup[d]_get_n_bytes() function to query the FFT setup handle.
  • To provide a buffer of the appropriate size to the FFT setup handle, the application can use the fft_setup[d]_set_buffer() function. The buffer is filled when the FFT setup handle is initialized.

The fft1d_setup_create() and fft1d_setupd_create() routines use an enumerated type argument, whose enumerators represent the individual inner loop routines. This argument refines and optimally configures the resources.

Note 
After it is initialized by fft_setup[d]_init(), the FFT setup handle can be passed only as an argument to the same FFT routine specified in the fft_type parameter.

log2n input

The base-2 exponent of the FFT size. The fft_setup[d]_init() function uses this argument to allocate storage for weights and index arrays required by the implementation. It also populates these arrays with weights and index values required by the specified FFT routine. In addition, the log2n value specified to fft1d_setup[d]_create() must be specified when invoking the actual FFT routine following initialization.

setup output

Allocates a setup handle that points to an opaque structure. This structure is used when the setup handle is initialized by fft_setup[d]_init() to complete the setup process by populating the FFT setup handle with (among other things) pointers to weights and index arrays

flag input

Reserved; specify SAL_RESERVED_FLAG.

Return Codes

SAL_INVALID_FUNCTION

SAL_FFT_NOT_SUPPORTED

An invalid enumerator was passed to the function.

SAL_INVALID_FLAG

An invalid flag was passed to the function.

SAL_INVALID_HANDLE

An invalid or null handle pointer was passed to the function.

SAL_INVALID_ARGUMENT

An invalid argument was passed to the function.

SAL_NOT_ENOUGH_MEMORY

Memory was insufficient for the function to perform all of its operations.

SAL_FAILURE

Function execution failed due to an unknown/uncategorized error.
