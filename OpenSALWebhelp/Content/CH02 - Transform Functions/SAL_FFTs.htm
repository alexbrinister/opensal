SAL Fast Fourier Transforms

SAL provides functions for single and multiple one-dimensional and two-dimensional transforms, and for mixed data types such as real-to-complex and complex-to-real transforms.

Except for inverse real FFTs, FFT (real or complex) output is a complex sequence.

FFT Features and Naming Convention

The SAL FFT function features include:

  • Radix
  • FFT type
  • Data type
  • In-place processing
  • Temporary buffer usage
  • Double precision

FFT function names are structured to represent these features. The naming structure is:

fft[3|5|9][2d|m]_{c|z|r|zr}{ip|op}[t][d]x(),

Where:

Element Meaning
3 radix-3 FFT (size of an FFT decomposition)
5 radix-5 FFT
9 radix-9 FFT
2d 2-dimensional FFT type
m multiple
c complex interleaved data type
z complex split
r real
zr odd/even real split
ip in-place (in/out-of-place)
op out-of-place
t with temporary buffer
d double precision

For example:

  • fft_zroptdx()—a 1-D, out-of-place, split real FFT with a temporary buffer
  • fft5m_copx()—a multiple, out-of-place, interleaved complex radix-5 FFT

Note 
Unless otherwise stated, all SAL functions support in-place execution.

Out-of-place functions with temporary buffers (those with fft_*opt[d]x() names) must have three cache buffers of equal size: one for input, one for output, and one temp buffer. Only the out-of-place functions with temporary buffers preserve input data. For more information, see FFT Memory

FFT Weights Arrays

SAL Fourier transforms (and other algorithms that process frequency/domain data) expect an array of complex exponentials to exist prior to calling the function. These complex exponentials are called FFT weights arrays (or sometimes twiddle factors). Once created, these arrays can be reused and shared by the same FFT function with the same size.

The weights array size (required to perform a given FFT) depends on the size of the FFT being calculated. The weights array size must be equal to the FFT size.

Creating Weights Arrays

To create an FFT weights array, you must call the fft1d_setup() or fft2d_setup() function (specifying the name of the FFT function you intend to call and the array size) at least once before you call any FFT or fast convolution and correlation (FC) function. The required size is the size of the FFT that will use this weights array for subsequent calculations.

The fft1d_setup() and fft2d_setup() functions perform the following tasks:

  • Allocates storage for the weights array (as an SMB).
  • Populates the array with the complex exponentials.

  • Returns an array descriptor in a structure allocated by the caller.

    You can pass the returned array descriptor as an FFT function argument.

These tasks also apply to the double-precision variants, fft1d_setupd() and fft2d_setupd().

The Setup Function n Argument

The weights array represents complex unit circle divisions. The setup function n argument is the base-2 logarithm of the number of divisions. This number specifies the number of elements that a subsequent FFT or FC function can process.

The setup function n value must be equal to the n value supplied to any FFT function using the weights array. Weights arrays generated for FC functions must be exactly equal to the size of the convolution and correlation. For more information, see Fast Convolution and Correlation Setup .

For instance, if the application calls the fft_ripx() function:

fft_ripx(&table, c, 1, 11, FFT_FORWARD, 0); /* 2048 (2**11) points */

you must call the fft1d_setup() function with an n value 11:

fft1d_setup(SAL_FFT_RIPX, 11, SAL_RESERVED_FLAG, &table); /* supports 2048 points */

FFT Memory

Depending on the features, different types of FFT functions require different amounts of memory. You might have to allocate properly aligned memory for the required FFT input, output, and temporary buffers. Memory allocations should be aligned on 16-byte boundaries.

Note 
FFT memory is not the same as FFT weights memory. For information on FFT weights memory allocation, see FFT Weights Arrays.

Packing Real Fourier Transforms

Real Fourier transforms have a real-valued input sequence and generally produce a complex-valued output sequence that is always conjugate-symmetric. Similarly, inverse real FFTs accept a conjugate-symmetric complex input sequence and produce a real output sequence.

Because of the conjugate symmetry of the complex sequence, only one half of that sequence is defined. Real-to-complex and double-to-double-complex discrete Fourier transforms write their output data in special packed formats. This way, the complex output requires no more memory than the real input.

Packing 1-D Real Fourier Transforms

Due to complex conjugate inherent symmetry, the forward Fourier transform of N floating-point inputs (real numbers) from the time domain to the frequency domain produces N/2+1 unique complex outputs. When the points are numbered from 0 to N-1, point N/2-i equals the N/2+i point complex conjugate. Therefore, to preserve the original information, only the first half of the frequency data, up to and including N/2+1, must be output.

In addition, SAL FFT takes advantage of the first complex output imaginary parts (element zero) and the last complex output (element N/2), the Nyquist frequency, always being zero.

As a result, the real part of the last output (element N/2) is stored where the imaginary part of the first output would have been. The zero-valued imaginary parts of the first and last outputs are implied.

Packing 2-D Real Fourier Transforms

When processing two-dimensional real inputs, the FFTs transform each row in the 2-D matrix and then transform columns, top to bottom.

Complex Interleaved Format

As each row is transformed, it is packed as previously described in Packing Real Fourier Transforms . For example, to transform an 8 x 8 real image to the frequency domain, each row is transformed and packed as shown in Figure 2-5.

Figure 2-5: Row Packing of an 8 x 8 Real-to-Complex 2-D FFT (Complex Interleaved)

Figure 2-5 represents an intermediate output after performing an 8-point interleaved complex FFT on each of the 8 x 8 matrix rows.

After the 1-D transform of each matrix row, the result is packed like the one-dimensional transform shown in Figure 2-3. Packing yields 16 real values and 24 complex values in a total of 64 sizeof(float) memory locations, the size of the original matrix.

After each row is processed and packed, each column FFT is calculated. Because the first pass left real values in columns zero and one, these two columns are transformed from real to complex using the same packing method that was used horizontally on each row.

Therefore, transforming these 16 real values yields six complex points and four real points, occupying the 16 memory locations in columns zero and one.

The remaining three columns of complex values are transformed complex-to-complex, yielding 24 complex points as output, and requiring 48 memory locations to store. This brings the total storage requirement to 64 locations, the size of the original matrix. The completed 8 x 8 transform format for the interleaved complex case is shown in Figure 2-6.

Conjugate Symmetry of Real Fourier Transforms

In Figure 2-6 (complex interleaved format) and Figure 2-7 (complex split format), only columns 1 through (N/2-1) are in true interleaved or split formats. The elements in columns zero and N/2 are in a special format. (The fft2d_[z]r{i|o}p[t]x() functions use this format for 2-D real FFTs.)

In both cases, there are a total of N*N = N2 float elements. However, the actual N x N real matrix 2-D FFT results in a total of N2 complex elements, or 2N2 float elements; twice the number shown in Figure 2-6 or Figure 2-7.

Only half of a real vector or matrix transform elements are calculated because of the conjugate symmetry of real transforms. The following equations describes an M x N matrix 2-D real FFT symmetry relationship.

Example 2-1: A M x N matrix 2-D real FFT symmetry relationship

If (m,n) is an M x N matrix of real numbers, where:

m=0,1,…,(M-1) and n=0,1,…,(N-1)

 

and F(j,k) is the 2-D FFT of ƒ(m,n), where:

j=0,1,…,(M-1) and k=0,1,…,(N-1)

 
Then, given that the elements of ƒ(m,n) are all real, F(j,k) has the following properties:

F(j,k) = F*(M–j,N–k)

(1)

where the asterisk (*) means complex conjugate.

It is also true of Fourier transforms in general that:

F(j,k) = F(j+M,k+N)

(2)

If j=0 or j=M/2 and k=0 or k=N/2, then it follows from Equation (1) and Equation (2) that F(0,0), F(0,N/2), F(M/2,0), and F(M/2,N/2) are all real numbers (that is, equal to their complex conjugates).

Another way to illustrate the idea of conjugate symmetry is to assume j=M/2+p and k=N/2+q, where -M/2 ≤ p < M/2 and -N/2 ≤ q < N/2. Substituting this into Equation (1) produces Equation (3).

F(M/2+p,N/2+q) = F*(M/2–p,N/2–q)

(3)

This proves conjugate symmetry of a real FFT about j=M/2 and k=N/2. Therefore, it is sufficient to calculate one-half of the total number of complex points in the transform plus the (0,0), (0,N/2), (M/2,0), and (M/2,N/2) points. The complex conjugates of the resulting values then immediately define the values of the other half of the transform.

The entire 2-D transform consists of M×N points, four of which ([0,0], [0,N/2], [M/2,] and [M/2,N/2]) are real. The remaining (M*N-4) points are typically complex. One-half of these (M×N-4) complex points must be calculated. You can find the other half using Equation (1) or Equation (3).

In addition, because of the real-valued 2-D array FFT symmetry, the values for k=0 and k=N/2 must be calculated for values of j=0,1,…,M/2 only. For these values of k, the transform values for the remaining values of j=(M/2+1),…,(M-1) again can be found using the conjugate symmetry principle expressed in Equation (1) or Equation (3).

For convenience when working with transformed images, SAL provides cimulx() and zimulx() functions that multiply packed frequency-domain images. These functions perform element-by-element complex multiplication of 2-D packed Fourier transforms of a real-valued 2-D matrix in complex interleaved (cimulx()) and complex split (zimulx()) formats. To perform a fast 2-D convolution of two 2-D image arrays, use these element-by-element multiplications of 2-D Fourier transforms. In other words, forward 2-D FFT, element-by-element complex multiplication by the second 2-D FFT, and calculation of the inverse 2-D FFT of the resulting matrix product.

Using complex one-dimensional transforms as an example, when transforming data from the time domain to the frequency domain with fft_copx(), no scaling factor is introduced according to Table 2-4. However, inverse transforms of frequency domain vectors from the frequency domain to the time domain with fft_copx() introduce a factor of N, where N is the vector length.

In Example 2-3:

  1. A vector is transformed to the frequency domain
  2. An inverse transform reconstructs it in the time domain
  3. It is scaled by 1/N to recover the original input (perhaps altered slightly by machine rounding error)

Example 2-3: Transform scaled by 1/N

fft.scale = 1.0/((float) FFT_LENGTH);
