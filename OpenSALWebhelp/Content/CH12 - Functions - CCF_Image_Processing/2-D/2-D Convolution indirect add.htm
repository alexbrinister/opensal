Description

The 2-D convolution _iax functions provide an alternate method of representing input matrices operated on by SAL convolution routines. Each matrix row is represented by a pointer stored in an array. To access any element of a particular matrix row, the pointer is read from the array and the element accessed by adding an offset to the pointer. For example, to read the element of an array from the second row, fourth column:

Real

SAL_f32 **row_array, *mptr, m24;

mptr = *(row_array + 2); /* read element M[2][4] */

m24 = *(mptr + 4);

SAL_f64 **row_array, *mptr, m24;

mptr = *(row_array + 2); /* read element M[2][4] */

m24 = *(mptr + 4);

Split Complex

SAL_zf32 Rp[16];

SAL_f32 *rmptr, *imptr, rm24, im24;

rmptr = Rp[2].realp;

imptr = Rp[2].imagp;

rm24 = *(rmptr + 4);

im24 = *(imptr + 4);

SAL_zf64 Rp[16];

SAL_f64 *rmptr, *imptr, rm24, im24;

rmptr = Rp[2].realp;

imptr = Rp[2].imagp;

rm24 = *(rmptr + 4);

im24 = *(imptr + 4);
