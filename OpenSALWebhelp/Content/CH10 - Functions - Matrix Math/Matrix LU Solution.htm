Matrix LU Solution

mat_lud_solx(), mat_lud_soldx()

Syntax

SAL_i32 mat_lud_solx(
  SAL_f32 *LU, /* input packed LU factor matrix */
  SAL_i32 LU_tcols, /* matrix LU column stride */
  SAL_i32 *P, /* permutation vector */
  SAL_f32 *b, /* right hand side vector, input */
  SAL_f32 *x, /* solution vector, output */
  SAL_i32 n, /* row/col.count for LU, elem.count for P, b, and x */
  SAL_i32 flag, /* transpose flag */
  SAL_i32 eflag ); /* ESAL flag */
SAL_i32 mat_lud_soldx(
  SAL_f64 *LU, /* input packed LU factor matrix */
  SAL_i32 LU_tcols, /* matrix LU column stride */
  SAL_i32 *P, /* permutation vector */
  SAL_f64 *b, /* right hand side vector, input */
  SAL_f64 *x, /* solution vector, output */
  SAL_i32 n, /* row/col.count for LU, elem.count for P, b, and x */
  SAL_i32 flag, /* transpose flag */
  SAL_i32 eflag ); /* ESAL flag */

Description

The mat_lud_solx() and mat_lud_solx() functions take as input:

  • An LU-packed matrix created by the matinvx(), matinvdx(), and matinv_lux() LU decomposition functions
  • A right-hand side vector b

As an output, this function provides a solution vector x for the equation A · x = b.

Here A = L · U, or a permutation thereof, with reciprocal diagonal elements in U. That is:

AP[i],j = (L · U)i,j for i ≠ j

where P is a permutation vector as output by the LU decompositions, and AP[i],i = 1/(L · U)i,i.

In addition, this solver routine can solve a system where the coefficient matrix is transposed (Hermitian conjugate in the complex case), AH · x = b. The ability to switch between the normal (nontransposed) and transposed cases using only the LU decomposition of A, without computing a separate LU decomposition for AH, is useful for solving systems, one with coefficient matrix of A and the other with coefficient matrix of AH.

The transposed case is different from the normal case, because if AP[i],j = (L · U)i,j is an LU decomposition in standard form (row-pivoted, unit lower triangular), then the transpose of the equation, AHj,P[i] = (UH · LH)i,j, is not in the standard form; it is in column-pivoted, unit upper triangular form. So, the flag variable is a means of selecting between performing the standard forward/back solve procedure and performing the non-standard (column-pivoted, unit upper triangular) variant of the procedure.

Parameters

LU

Input matrix containing triangular LU factors of a matrix A in the special packed format described in LU Decomposition. To obtain the input, use the matinvx(), matinvdx(), and matinv_lux() functions.

LU_tcols

Column element stride for LU.

P

Permutation vector created by the matinvx(), matinvdx(), and matinv_lux() functions.

b

Right hand side vector. Input.

x

Solution vector. Output.

n

Row and column count for LU, element count for D, x, and b.

flag

Specify one of the following transpose options:

SAL_NORMAL_SOLVER

Solve A · x = b. (default)

SAL_TRANSPOSE_SOLVER

Solve AH · x = b.

Returns

SAL_SUCCESS

Successful execution.

Other return values

A floating point exception is detected.
