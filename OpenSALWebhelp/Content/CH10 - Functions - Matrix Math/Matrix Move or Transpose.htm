Matrix or Submatrix Move or Transpose

mmovx(), mmovdx()

Syntax

void mmovx(
  SAL_f32 *A, /* input matrix */
  SAL_i32 ax, /* x-stride for A */
  SAL_i32 ay, /* y-stride for A */
  SAL_f32 *C, /* output matrix */
  SAL_i32 cx, /* x-stride for C */
  SAL_i32 cy, /* y-stride for C */
  SAL_i32 X, /* element count in x-dimension of A and C */
  SAL_i32 Y, /* element count in y-dimension of A and C */
  SAL_i32 eflag ); /* ESAL flag */
void mmovdx(
  SAL_f64 *A, /* input matrix */
  SAL_i32 ax, /* x-stride for A */
  SAL_i32 ay, /* y-stride for A */
  SAL_f64 *C, /* output matrix */
  SAL_i32 cx, /* x-stride for C */
  SAL_i32 cy, /* y-stride for C */
  SAL_i32 X, /* element count in x-dimension of A and C */
  SAL_i32 Y, /* element count in y-dimension of A and C */
  SAL_i32 eflag ); /* ESAL flag */

Description

The mmovx() and mmovdx() submatrix move functions move or transpose matrices or submatrices. These operations can be done out-of-place only.

Parameters

*A

Pointer to the start of the input submatrix.

ax, ay

Row and column strides for matrix A. To indicate a certain frequently-used input matrix configuration, ou can specify zero for either of the input matrix strides (but not both). The value ax = 0 is interpreted as ax = Y * ay, and the submatrix y-dimension is the entire input matrix y-dimension. The value ay = 0 is interpreted as ay = X * ax, and the submatrix x-dimension is the entire input matrix x-dimension.

*C

Pointer to the start of the output matrix.

cx, cy

Row and column strides for matrix C. To indicate a certain frequently-used output matrix configuration, you can specify zero for either of the output matrix strides (but not both). The value cx = 0 is interpreted as cx = Y * cy, and the submatrix y-dimension is the entire output matrix y-dimension. The value cy = 0 is interpreted as cy = X * cx, and the submatrix x-dimension is the entire output x-dimension width.

X

Number of columns to move from A to C.

Y

Number of rows to move from A to C.

To avoid non-unique element indexing, make sure that in both of the following pairs at least one of two conditions is satisfied:

ax >= ay * Y or ay >= ax * X

cx >= cy * Y or cy >= cx * X

Examples

Strides for complex interleaved data are counted in half-elements. Therefore, similar calls appear different in complex split and interleaved routines.

Example 9-1: Typical matrix move parameters

ax = cx = 1;

ay = cy = X;

mmovx ( A, 1, 500, C, 1, 500, 500, 600, SAL_CC);

Example 9-2: Typical matrix transpose parameters

ax = cy = 1;

ay = X;

cx = Y;

mmovx (A, 1, 500, C, 600, 1, 500, 600, SAL_CC);

- or -

mmovx (A, 500, 1, C, 1, 600, 600, 500, SAL_CC);

For a more direct approach to matrix transposition, use mtransx() and mtransdx().

Example 9-3: Typical submatrix move parameters

ax = cx = 1;

ay = cy = XX; /* XX > X */

mmovx ( A, 1, 1000, C, 1, 1200, 500, 600, SAL_CC);

Example 9-4: Typical submatrix transpose parameters

ax = cy = 1;

ay = XX; /* XX > X */

cx = YY; /* YY > Y */

mmovx (A, 1, 1000, C, 1200, 1, 500, 600, SAL_CC);

- or -

mmovx (A, 1000, 1, C, 1, 1200, 600, 500, SAL_CC);

For a more direct approach to submatrix transposition, use mtrans2x() and mtrans2dx().
